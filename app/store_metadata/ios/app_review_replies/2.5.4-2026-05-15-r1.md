## Reply body (paste into App Store Connect)

Hello,

Thank you for the detailed review of submission 2f88a54d-2b9a-4069-b5fa-88e2ed770187 and for flagging the 2.5.4 concern. We agree that persistent background location should serve the user themselves, not just their employer. In build 0.3.1 (8) we have added an end-user-facing feature that uses the same background-collected location data, so the AppUser is the primary beneficiary of the `UIBackgroundModes: location` entitlement.

**Where to find the new feature**

1. After logging in with the demo credentials below, you will land on the home tab. A new **"我的今天 / Today"** card appears between the clock-in / out buttons and the queue chip. It shows the user their own walked distance and on-shift duration for the current day, updated live while the shift is active.
2. The bottom navigation bar now has a third tab labeled **"我的軌跡 / My Trajectory"**. Tapping it opens **"我的工作日記 / My Work Day"** — a per-day map polyline of the AppUser's own movement during their shift, with summary stats (distance, on-shift duration, number of location samples). The user can pick today or any of the previous 7 days from a dropdown.
3. The home-screen card is tappable; tapping it deep-links into the same trajectory screen with today selected.

The trajectory tab and the home card are both reachable in one tap from any authenticated screen — they are not hidden behind a settings or overflow menu.

**App Privacy form update**

We have added "App Functionality" as a use case for Precise Location alongside the existing "Other Purposes" use case, reflecting that the AppUser themselves now consumes the location data directly through "我的工作日記" in addition to the org-side admin export.

**Consent / permission description**

The `NSLocationWhenInUseUsageDescription` string and the in-app clock-in consent dialog have both been reworded to lead with the AppUser-facing benefit ("讓你可以在 app 內回顧自己每一天的工作路線與走動距離") before mentioning that admins can also view the data, so the message presented at permission time matches the rest of the experience.

We would be grateful if you could re-run the review against build 0.3.1 (8). If anything is unclear about how to reach the new tab or card we are happy to record a short screen capture and attach it to this thread.

Thank you again,

The Bandao team
support@ccmos.tw
